## US-1 建立相簿並整理照片（優先級：P1）

### Scenario: S-1-1 建立「旅行」相簿並一次加入五張 JPG

```gherkin
Scenario: 建立「旅行」相簿並一次加入五張 JPG
  Given 使用者尚未建立任何相簿
  And 本機有至少五張可選的 JPG 照片
  When 使用者建立一個名為「旅行」的相簿並一次將這五張 JPG 照片加入「旅行」相簿
  Then 系統顯示「旅行」相簿
  And 該相簿內含這五張照片
```

**對應欄位**:

- US-1 建立相簿並整理照片（優先級：P1）— 使用者可以建立多個相簿，將照片放入對應相簿中，讓原本零散的照片依主題或用途被整理起來。
- AC-1-1 假設使用者尚未建立任何相簿，當使用者建立一個名為「旅行」的相簿並一次加入 5 張 JPG 照片，則系統應顯示「旅行」相簿且內含這 5 張照片
- US1-FR1 系統必須允許使用者建立相簿並命名
- US1-FR2 系統必須允許使用者將照片加入指定相簿
- US1-FR3 系統必須支援一次選取多張照片匯入，且支援 JPG、PNG、HEIC 等常見格式

## US-2 在主頁面依日期瀏覽相簿（優先級：P2）

### Scenario: S-2-1 主頁依建立日期分組顯示相簿

```gherkin
Scenario: 主頁依建立日期分組顯示相簿
  Given 使用者已建立多個相簿
  And 各相簿具有不同的建立日期
  When 使用者進入主頁面
  Then 系統依相簿建立日期將相簿分組顯示
```

**對應欄位**:

- US-2 在主頁面依日期瀏覽相簿（優先級：P2）— 使用者回到主頁面時，可以看到所有相簿依建立日期分組顯示，藉此更快找到特定時間相關的照片集合。
- AC-2-1 假設使用者已建立多個相簿且各自有不同的建立日期，當使用者進入主頁面，則系統應依相簿建立日期將相簿分組顯示
- US2-FR1 系統必須在主頁面顯示所有相簿
- US2-FR2 系統必須依相簿建立日期將相簿分組顯示

---

- Edge-2-1 當某個日期分組下沒有任何相簿時，系統不應顯示空白日期分組

## US-3 透過拖放重新排列相簿（優先級：P3）

### Scenario: S-3-1 拖放重排後顯示新順序並再次進入主頁仍保留

```gherkin
Scenario: 拖放重排後顯示新順序並再次進入主頁仍保留
  Given 主頁面上已有多個相簿
  When 使用者將其中一個相簿拖放到新的位置並再次進入主頁面
  Then 系統依新的順序顯示相簿
  And 重新整理或再次進入主頁面後仍保留該順序
```

**對應欄位**:

- US-3 透過拖放重新排列相簿（優先級：P3）— 使用者可以在主頁面直接拖放相簿，調整自己想要的排列順序，讓常用或重要的相簿排在更顯眼的位置。
- AC-3-1 假設主頁面上已有多個相簿，當使用者將其中一個相簿拖放到新的位置，則系統應依新的順序顯示相簿
- US3-FR1 使用者必須能夠在主頁面透過拖放重新排列相簿順序
- US3-FR2 系統必須保存使用者重新排列後的相簿順序

---

Scenario: 僅一個相簿時拖放不造成錯誤或混淆

  Then 系統不造成錯誤或混淆的互動結果

- Edge-3-1 當主頁面只有一個相簿時，拖放操作不應造成錯誤或混淆的互動結果
- US3-FR1 使用者必須能夠在主頁面透過拖放重新排列相簿順序

## US-4 在相簿內以平鋪方式預覽照片（優先級：P3）

### Scenario: S-4-1 有照片的相簿以平鋪預覽顯示

```gherkin
Scenario: 有照片的相簿以平鋪預覽顯示
  Given 某個相簿內已有多張照片
  When 使用者打開該相簿
  Then 系統以平鋪式預覽顯示相簿中的照片
```

**對應欄位**:

- US-4 在相簿內以平鋪方式預覽照片（優先級：P3）— 使用者進入任一相簿後，可以一次看到多張照片的縮圖平鋪排列，快速掃描相簿內容。
- AC-4-1 假設某個相簿內已有多張照片，當使用者打開該相簿，則系統應以平鋪式預覽顯示相簿中的照片
- US4-FR1 系統必須允許使用者進入單一相簿查看內容
- US4-FR2 系統必須在相簿內以平鋪式介面預覽照片

---

### Scenario: S-4-2 空相簿顯示可理解的空狀態

```gherkin
Scenario: 空相簿顯示可理解的空狀態
  Given 某個相簿內尚無照片
  When 使用者打開該相簿
  Then 系統顯示可理解的空狀態
  And 不是空白畫面
```

**對應欄位**:

- US-4 在相簿內以平鋪方式預覽照片（優先級：P3）— 使用者進入任一相簿後，可以一次看到多張照片的縮圖平鋪排列，快速掃描相簿內容。
- Edge-4-1 當相簿內尚無照片時，系統應顯示可理解的空狀態，而不是空白畫面
- US4-FR1 系統必須允許使用者進入單一相簿查看內容
- US4-FR3 系統必須在相簿沒有照片時顯示可理解的空狀態

## 假設

- 本計畫只依同 package 的 `spec.md` 展開；GWT 使用領域語言，不綁定具體 API 或頁面操作實作，亦不讀取 `system-analyze/`。
- Gherkin 慣例：`Given`／`Then` 可用 `And` 延續；`When` 僅一條操作步驟（不在 When 後接 And）。
- AC 與 Edge 語意重複時合併為單一 Scenario；本期無此合併項。
- 本期無含 `NEED CLARIFICATION` 的邊界，故無 blocked Scenario。
- Edge ID（Edge-x-y）對應各 US「邊界情境」條列順序；`Edge-G-1` 對應「全域邊界情境」唯一一條（相簿不可巢狀）。
- S-3-1 在 AC-3-1「立即依新順序顯示」之外，依 US3-FR2 與獨立驗證方式一併驗收「再次進入主頁後仍保留順序」。
- FR 編號沿用 `spec.md` 原文（如 `US1-FR1`），不以另編 `FR-001` 取代。
