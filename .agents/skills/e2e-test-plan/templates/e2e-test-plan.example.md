## US-1 手動建立相簿並在相簿內加入照片（優先級：P1）

### Scenario: S-1-1 建立「旅行」相簿並加入一張照片

```gherkin
Scenario: 建立「旅行」相簿並加入一張照片
  Given 使用者尚未建立任何相簿
  And 本機有至少一張可選的照片
  When 使用者建立一個名為「旅行」的相簿並從本機將該照片加入「旅行」相簿
  Then 「旅行」相簿存在
  And 該相簿內含剛加入的那張照片
```

**對應欄位**:

- US-1 手動建立相簿並在相簿內加入照片（優先級：P1）— 本機使用者想建立自己的相簿，並直接在相簿內加入本機照片，完成「把照片整理進相簿」的核心動作。
- AC-1-1 假設使用者尚未建立任何相簿，當建立一個名為「旅行」的相簿並從本機加入一張照片，則該相簿存在且內含該照片
- FR-001 系統必須允許使用者建立新的相簿，並為相簿指定可辨識的名稱
- FR-002 系統必須允許使用者進入某個相簿後，從本機選擇照片加入該相簿
- FR-004 使用者完成加入後，系統必須讓使用者能確認照片已進入該相簿

### Scenario: S-1-2 拒絕相簿巢狀並維持單層

```gherkin

  Given 使用者已在某個相簿中
  And 系統中已存在至少另一個相簿
  When 使用者嘗試在目前相簿下建立子相簿，或把另一個相簿放進目前相簿
  Then 系統拒絕巢狀結構
  And 所有相簿仍維持單層
```

**對應欄位**:

- US-1 手動建立相簿並在相簿內加入照片（優先級：P1）— 本機使用者想建立自己的相簿，並直接在相簿內加入本機照片，完成「把照片整理進相簿」的核心動作。
- AC-1-2 假設使用者已在某個相簿中，當嘗試建立子相簿或把另一個相簿拖進目前相簿，則系統拒絕巢狀結構並維持所有相簿為單層
- Edge-1-1 當使用者嘗試在相簿下再建立子相簿時，系統必須阻止並維持單層結構
- FR-003 系統必須確保相簿只能以單層結構存在，不可把相簿放進另一個相簿

## US-4 從照片庫將照片分配到相簿（優先級：P4）



### Scenario: S-4-1 匯入照片到照片庫後可查看與選取

```gherkin
Scenario: 匯入照片到照片庫後可查看與選取
  Given 使用者已有至少一個相簿

```

**對應欄位**:

- US-4 從照片庫將照片分配到相簿（優先級：P4）— 使用者想先把照片匯入照片庫，再從照片庫把照片分配到一個或多個相簿，而不必每次都只從相簿內加入。
- AC-4-1 假設使用者已有至少一個相簿，當從本機匯入照片到照片庫，則這些照片可在照片庫中被查看與選取
- FR-011 系統必須提供照片庫，讓使用者可匯入並查看尚未或不限於特定相簿的本機照片

## US-5 依日期自動產生相簿（優先級：P5）



### Scenario: S-5-1 依日期自動成冊並可在相簿中找到照片

```gherkin
Scenario: 依日期自動成冊並可在相簿中找到照片
  Given 使用者新增多張具有明確日期的照片
  When 系統執行依日期自動成冊
  Then 出現對應日期的相簿
  And 這些照片可在對應相簿中被找到
```

**對應欄位**:

- US-5 依日期自動產生相簿（優先級：P5）— 使用者希望系統能依照片日期自動產生相簿，減少手動建冊的負擔，並與手動建立的相簿並存於主頁。
- AC-5-1 假設使用者新增多張具有明確日期的照片，當系統執行依日期自動成冊，則會出現對應日期的相簿，且照片可在其中被找到
- FR-014 系統必須能依照片的日期資訊，自動產生對應的相簿
- FR-016 當照片具備可用日期資訊時，系統必須把該照片歸入對應的自動日期相簿
