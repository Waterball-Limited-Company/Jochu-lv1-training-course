# SOP

## Phase 1 -- 讀取目標

1. READ 讀取目標 RuleFile 內容，確認要新增、修改或重寫的規則範圍。

## Phase 2 -- 按需思考並完成

1. THINK 在思考 Rule 結構時，按需讀取 `rules/RuleLevel-定義.md` 與 `rules/RuleFile-格式規範.md`；只在需要判斷 `Level`、欄位順序、Example 邊界或其他格式限制時才載入，並依實際讀取到的規則收斂最小可用結構。
2. WRITE 依思考結果撰寫或修改目標 RuleFile。
3. READ 回頭檢查結果是否符合本次實際載入的規則；若不符合，立即修正。
