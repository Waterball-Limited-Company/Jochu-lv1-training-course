# Rule 1 - 高影響缺口必須先澄清，或在正文標 `[NEEDS CLARIFICATION]`

- Level: `MUST`
- 若缺口會改變資源切分、Endpoint 集合、成功／錯誤 status 語意、跨實體所有權，或會讓多個 Endpoint 契約同時改寫，優先呼叫 `/clarify`；未拍板前不可寫成看似已確認的契約。
- 若必須先寫暫定契約供審閱，必須在受影響的 Endpoint／設計備註等正文內嵌 `[NEEDS CLARIFICATION: …]`，並可在檔末 `## 假設` 註明第一版暫定取捨。
- 標記密度對齊 specify：只標高影響缺口；低風險預設不標標記。

## Good Example

- 這個例子是好的，因為會改多個 Endpoint 的規則在正文有標記。

```md
| 設計備註 | … [NEEDS CLARIFICATION: 需確認分配到相簿後照片是否仍留在照片庫] |
```

## Bad Example

- 這個例子是壞的，因為主契約未定卻寫死所有 Endpoint 且無標記。

```md
先假設允許多相簿歸屬，把所有 Endpoint 都寫成可重複 assign，之後有空再問。
```

# Rule 2 - `[NEEDS CLARIFICATION]` 格式必須與 specify 一致

- Level: `MUST`
- 待澄清標記固定為 `[NEEDS CLARIFICATION: …]`（`NEEDS` 複數），必須明說缺的決策軸。
- 不可使用 `[Need Clarification]`、`[NEED CLARIFICATION]` 等變體作為契約標記。

## Good Example

- 這個例子是好的，因為決策軸清楚。

```md
[NEEDS CLARIFICATION: 需確認分配到相簿後照片是否仍留在照片庫]
```

## Bad Example

- 這個例子是壞的，因為字串不一致且語意含糊。

```md
[Need Clarification: API 細節之後補]
```

# Rule 3 - 低風險不確定性只寫檔末「假設」，不打待澄清標記

- Level: `MUST`
- 若缺口不致改變 api-plan 主結構，可用第一版假設繼續，寫入檔末 `## 假設`（`- ` 條列）。
- 此類低風險項目**不要**使用 `[NEEDS CLARIFICATION]`。
- 澄清後必須回寫 `## 假設` 與相關 Endpoint。

## Good Example

- 這個例子是好的，因為低風險只進假設。

```md
## 假設

- 列表預設排序固定為 created_at 升冪，第一版不提供排序 query
```

## Bad Example

- 這個例子是壞的，因為把預設排序偏好也打成待澄清標記。

```md
[NEEDS CLARIFICATION: 列表要不要加 sort query]
```

# Rule 4 - 檔案最下方必須有與 spec.md 同格式的「假設」區塊

- Level: `MUST`
- `api-plan.md` 必須在檔案最下方提供 `## 假設`，以 `- ` 條列全域前提與本檔第一版設計假設。
- 對已標 `[NEEDS CLARIFICATION]` 且採暫定往前的項目，應在假設中交代暫定取捨。

## Good Example

- 這個例子是好的，因為假設在檔末。

```md
## 假設

- 庫內分配後是否仍留庫：正文已標 NEEDS CLARIFICATION；第一版暫定仍留庫
```

## Bad Example

- 這個例子是壞的，因為缺少檔末假設區塊。

```md
## 實體: Photo

（檔案結束）
```

# Rule 5 - 上游 `[NEEDS CLARIFICATION]` 必須被消化，不可默默消失

- Level: `SHOULD`
- 對 `spec.md` 中與 API 合約相關的 `[NEEDS CLARIFICATION]`，應在本輪澄清、在本檔正文保留／改寫對應標記、寫入 `## 假設`，或聲明「本期不影響 API／延後」。

## Good Example

- 這個例子是好的，因為上游缺口都有去向。

```md
spec 缺口 A → 已 clarify
spec 缺口 B → 本檔正文保留 [NEEDS CLARIFICATION: …] 並於 ## 假設註暫定
spec 缺口 C → ## 假設 註明不影響本期 API
```

## Bad Example

- 這個例子是壞的，因為上游缺口被默默丟掉。

```md
只寫有把握的 Endpoint，spec 裡的 NEEDS CLARIFICATION 一律不提。
```
