## SOP 格式規範

每份 skill 都必須包含 `# SOP` section。SOP 以 **Phase** 分階段、以 **Step** 描述可執行動作。

### Phase 標題

```md
## Phase N -- <階段標題>
```

- `N` 從 1 開始，依序遞增
- 階段標題簡短描述此 phase 的階段成果，而非抽象主題

### Step 格式

```md
1. <動詞> <描述此步驟所需進行的行為>
```

每個 step 以動詞開頭，動詞語意如下：

| 動詞 | 語意 |
|------|------|
| `READ` | 讀取檔案、文件、上下文 |
| `WRITE` | 寫入或修改檔案、產出內容 |
| `THINK` | 分析、拆解、判斷、收斂（extended thinking） |
| `DELEGATE` | 委派給 script、其他 skill 或 agent |

### 撰寫原則

- Phase 必須有順序，後一 phase 建立在前一 phase 的產出之上
- Step 必須可執行，避免空泛指示（如「注意品質」）
- 每一步寫清楚處理對象、動作、目的
- 需要委派時，用 `DELEGATE` 明確標示對象與交付內容
- 只保留真正影響結果的步驟，刪除重複或無法落地的描述
