# Rule 1 - 回寫原 step 時必須把新 RuleFile 的讀取時機與路徑寫進句子

- Level: `MUST`
- 開新 RuleFile 後，回寫原 step 時應在原句中明確寫出「什麼情況下」需要讀取該檔案，並直接放入路徑，如 `rules/xxx.md`。
- 路徑應嵌在 step 的動作句內，讓執行者知道先讀哪份 rule，再拿它做什麼。
- 不可只新增一句籠統提醒，例如「必要時參考相關規則」或「去看某份 rule」，而不交代讀取時機與後續動作。

## Good Example

- 這個例子是好的，因為它示範了開新 RuleFile 之後，如何把路徑與讀取時機直接嵌回原本的 step。

````md
原本：
2. THINK 根據該 step 的目的、輸入輸出與常見失誤，判斷哪些要求應抽成可按需載入的 RuleFile，哪些內容仍應留在原 step 中，避免把規則展開誤寫成流程改寫。

新增 `rules/規則展開邊界.md` 後：
2. THINK 若需要判斷哪些要求應抽成可按需載入的 RuleFile、哪些內容仍應留在原 step，先讀取 `rules/規則展開邊界.md`，再根據該 step 的目的、輸入輸出與常見失誤收斂規則邊界，避免把規則展開誤寫成流程改寫。
````

## Bad Example

- 這個例子是壞的，因為它雖然提到 rule，但沒有寫出何時讀、讀完後要怎麼用回原步驟。

````md
2. THINK 參考 `rules/規則展開邊界.md`。
````
