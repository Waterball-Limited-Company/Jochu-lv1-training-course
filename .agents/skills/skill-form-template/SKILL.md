# SOP

## Phase 1 -- 讀取目標

1. READ 讀取目標骨架檔與對應範例檔內容，確認要新增、修改或重寫的樣板範圍。

## Phase 2 -- 按需思考並完成

1. THINK 在思考樣板結構時，按需讀取 `rules/樣板雙檔配對.md` 與 `rules/填位符號格式規範.md`；只在需要判斷骨架與範例的分工、命名、副檔名、填位符號寫法或其他格式限制時才載入，並依實際讀取到的規則收斂最小可用樣板。
2. WRITE 依思考結果撰寫或修改目標骨架檔與對應範例檔；樣板中的章節標題、欄位名稱、註解、說明文字與範例內容皆必須以繁體中文為主（程式語法、固定識別 token、檔名與專有名詞除外）。
3. READ 回頭檢查結果是否符合本次實際載入的規則；若不符合，立即修正。
