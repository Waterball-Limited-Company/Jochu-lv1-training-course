# SOP

## Phase 1 -- 收斂樣板展開範圍

1. READ 讀取目標 SOP skill 內容，定位使用者指定的 phase 與 step，確認要展開的是哪個步驟的樣板。
2. THINK 若需要判斷哪些要求應抽成可按需讀取的樣板、哪些內容仍應留在原 step，先讀取 `rules/樣板抽離邊界.md`，再根據該 step 的目的、輸入輸出與預期產物收斂樣板邊界，避免把樣板抽離誤寫成流程改寫或規則展開。

## Phase 2 -- 展開樣板並掛回 SOP

1. DELEGATE 若已收斂出樣板範圍，呼叫 `/skill-form-template` 撰寫或修改目標 skill `templates/` 下對應的骨架檔與範例檔，交付該 step 需要複製與改寫的輸出樣板。
2. WRITE 回寫目標 SOP skill 的指定 step，補上何時按需讀取該骨架檔與範例檔的說明，讓主流程維持最小可用、樣板只在需要生成對應格式時才載入。
3. READ 若需要檢查回寫後的指定 step 是否已把新樣板的讀取時機與兩個檔案路徑嵌回原句，先讀取 `rules/回寫時同時掛骨架與範例.md`，再回頭檢查新樣板與回寫後的 SOP 是否一致：指定 step 仍描述當下動作、骨架檔與範例檔只在實際讀取時被點名，且流程已形成「先有 SOP、再逐步展開樣板」的結構；若不符合，立即修正。
