# Rule 1 - 回寫原 step 時必須把骨架檔與範例檔的讀取時機和兩條路徑寫進句子

- Level: `MUST`
- 開新樣板雙檔後，回寫原 step 時應在原句中明確寫出什麼情況下需要讀取該骨架檔與範例檔，並直接放入兩條路徑。
- 路徑必須同時包含骨架檔與範例檔，例如 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`。
- 不可只新增一句籠統提醒，例如「必要時參考相關樣板」，也不可只掛其中一個檔案。

## Good Example

- 這個例子是好的，因為它把讀取時機與兩個路徑直接嵌回原本的 step。

```md
2. WRITE 若需要產出類別圖，先讀取 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`，再依目前分析結果完成圖面內容。
```

## Bad Example

- 這個例子是壞的，因為它沒有交代完整路徑，也沒有說清楚何時要讀兩個檔案。

```md
2. WRITE 參考相關樣板產出類別圖。
```

# Rule 2 - 回寫句子必須說明讀完骨架與範例後要如何使用

- Level: `MUST`
- 回寫後的 step 應讓執行者知道骨架檔是用來複製結構，範例檔是用來對照完成品，再依目前上下文改寫填位符號或具體內容。
- 不可只列出檔名而不說明讀完之後的下一步，否則執行者仍不知道怎麼把雙檔用回當前任務。
- 若 step 需要特別指出先複製骨架再參考範例，應把這個使用順序寫進句子。

## Good Example

- 這個例子是好的，因為它不只列出兩個檔案，還說明了骨架與範例各自的用途。

```md
2. WRITE 若需要產出類別圖，先讀取 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`，再依骨架複製結構、參考範例改寫其中的填位符號與具體內容。
```

## Bad Example

- 這個例子是壞的，因為它雖然提到兩個檔案，但沒有交代讀完之後要如何使用它們。

```md
2. WRITE 若需要產出類別圖，先讀取 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`。
```

# Rule 3 - 回寫後的 step 仍必須保留當下產物與目的

- Level: `MUST`
- 即使已補上樣板雙檔路徑，step 仍必須保留當下要生成的產物名稱與這一步的直接目的。
- 不可讓回寫後的 step 退化成只剩讀檔與複製格式，卻沒有說明要對哪份內容完成什麼輸出。
- 若樣板被拿掉後，step 仍能看出當下要做的核心產物與動作，表示回寫位置合理。

## Good Example

- 這個例子是好的，因為它仍清楚交代這一步要產出類別圖，且要根據目前分析結果完成內容。

```md
2. WRITE 若需要產出類別圖，先讀取 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`，再依骨架複製結構、參考範例改寫其中的填位符號與具體內容，完成可直接交付的 Mermaid 類別圖。
```

## Bad Example

- 這個例子是壞的，因為它把重點縮成讀檔與套樣板，失去這一步本來要完成的產物指向。

```md
2. WRITE 先讀取 `templates/class-diagram.mmd` 與 `templates/class-diagram.example.mmd`，再照著改。
```
